# MealRo MVP 기능 우선순위

## 문서 목적

이 문서는 Figma 보드 `MealRo / node 204:694`를 기준으로, 현재 MVP가 어떤 사용자 흐름과 기능 우선순위로 설계되고 있는지 정리한 참고 문서다.

핵심 목적은 다음 두 가지다.

1. 아직 전체 화면과 디자인 시스템이 확정되지 않은 상태에서 MVP의 중심 가치를 흔들리지 않게 유지한다.
2. 앞으로의 리팩토링과 구현 순서를 제품 흐름 기준으로 정렬한다.

## Figma 기준 MVP 한 줄 정의

사용자가 식사 시간을 선택하고 음식을 촬영하면, AI가 현재 식사를 분석하고, 이후 추천 로직과 지도 정보를 통해 다음 식사 선택까지 연결해주는 모바일 식사 코치 앱.

## 핵심 제품 가설

- 사용자는 "뭘 먹을지"보다 먼저 "지금 먹은 것이 어떤 의미인지"를 알고 싶다.
- 분석 결과만 보여주는 앱보다, 다음 행동인 추천과 방문/선택까지 이어주는 앱이 더 강한 가치를 만든다.
- MVP 단계에서도 `분석 -> 추천 -> 실제 선택`의 연결이 중요하다.

## MVP 중심 사용자 흐름

### 1. 식사 입력 진입

- 홈 또는 하단 중앙 FAB를 통해 식사 입력 플로우로 진입
- 사용자는 `아침 / 점심 / 저녁 / 간식` 중 식사 시간을 선택

### 2. 촬영 및 분석

- 음식 사진을 촬영 또는 업로드
- AI가 음식 종류와 영양 정보를 분석
- 사용자는 칼로리, 주요 영양소, 요약 인사이트를 확인

### 3. 추천 연결

- 현재 식사 결과를 기반으로 다음 행동을 제안
- 보드상 추천 방향:
  - 역추산 알고리즘
  - 위치 기반 메뉴 추천
  - 사용자 식단 목적 기반 추천
  - 프랜차이즈/배달 메뉴 추천
  - 영업시간/가격/거리 고려

### 4. 지도 및 상세 탐색

- 추천 후보를 지도에서 확인
- 식당 또는 메뉴 상세 정보 확인
- 실제 방문/선택 흐름으로 이동

### 5. 분석/회고

- 식사 누적 결과를 요약 화면에서 확인
- 주간 또는 누적 기준으로 식단 관리를 계속하도록 유도

## 우선순위 정리

## P0. 반드시 먼저 살아야 하는 기능

이 단계가 MVP의 본체다.

- 식사 시간 선택
- 카메라/FAB 진입
- 음식 촬영 또는 업로드
- AI 분석 처리
- 분석 결과 화면
- 결과 저장 또는 다음 단계로 연결되는 CTA

### P0 성공 조건

- 사용자가 1분 안에 첫 분석 결과를 볼 수 있어야 한다.
- 분석 결과는 최소한 `음식명`, `칼로리`, `핵심 영양 정보`를 제공해야 한다.
- 분석 후 사용자가 다음 행동을 선택할 수 있어야 한다.

## P1. 분석 이후 가치를 완성하는 기능

이 단계가 MealRo를 단순 분석 앱이 아니라 추천 서비스로 보이게 만든다.

- 역추산 기반 식단 추천
- 사용자 목적 기반 추천
- 위치 기반 추천
- 식당/메뉴 리스트
- 지도와 추천 후보 연결
- 간단한 식당/메뉴 상세

### P1 성공 조건

- 분석 결과 이후 "그래서 다음엔 뭘 먹지?"에 대한 제안이 보여야 한다.
- 추천은 최소한 하나의 기준 축을 가져야 한다.
  - 예: 칼로리 균형, 사용자 목표, 거리, 가격

## P2. 유지/재방문을 만드는 기능

- 주간/월간 분석 대시보드
- 식사 이력
- 온보딩 기반 개인화 정교화
- 로그인 후 데이터 복구
- 커뮤니티/피드 확장

### P2 성공 조건

- 재방문 사용자가 자신의 누적 상태를 이해할 수 있어야 한다.
- 게스트 사용자가 계정 전환 시 기록 손실 없이 이어갈 수 있어야 한다.

## MVP 단계에서 제외하거나 약하게 가져가도 되는 것

다음 항목은 중요하지만, 현재 Figma 보드 기준으로도 핵심 본체보다 후순위다.

- 커뮤니티 피드의 정교한 상호작용
- 고도화된 소셜 기능
- 정밀 차트 시스템
- 복잡한 추천 필터링 UX
- 완성도 높은 브랜드 모션 시스템
- 다중 추천 알고리즘 실험판 동시 운영

## 기능 묶음 기준

앞으로 구현과 리팩토링은 아래 묶음으로 진행하는 것이 합리적이다.

### 묶음 A. Capture

- 식사 시간 선택
- 카메라 진입
- 업로드/촬영
- AI 분석

### 묶음 B. Decision

- 분석 결과
- 추천 로직
- 저장/다음 행동 CTA

### 묶음 C. Explore

- 지도
- 추천 리스트
- 상세
- 방문 확인

### 묶음 D. Retention

- 분석 대시보드
- 이력
- 계정 전환/복구

## 현재 리팩토링에 주는 의미

이 Figma 보드는 모든 화면을 확정해주는 문서가 아니라, 제품의 우선순위를 보여주는 문서다.

따라서 지금 리팩토링의 기준은 다음이어야 한다.

- 첫 번째로 `Capture + Decision` 흐름을 가장 강하게 다룬다.
- 두 번째로 `Explore`를 연결 가능한 구조로 만든다.
- `Retention`은 MVP 흐름을 방해하지 않는 범위에서 정리한다.

## 결론

MealRo MVP의 본질은 "음식 분석 앱"이 아니라, `지금 먹은 식사 이해 -> 다음 식사 결정`까지 이어주는 식사 코치 경험이다.

따라서 구현 우선순위는 아래 순서로 고정하는 것이 맞다.

1. 촬영/분석
2. 분석 결과와 추천 연결
3. 지도/상세/실제 선택
4. 누적 분석과 재방문 경험
