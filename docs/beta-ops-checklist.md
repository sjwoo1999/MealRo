# MealRo Beta Ops Checklist

이 문서는 현재 브랜치 기준 MealRo를 베타 테스트로 배포하거나 공유하기 전에 확인해야 할 운영 체크리스트다.

## 1. 제품 상태 확인

- 홈에서 카메라 중심 진입과 식사 선택이 바로 가능한지 확인
- `/scan`에서 업로드, 분석, 수정, 저장이 가능한지 확인
- `/meal`에서 기준 메뉴 검색과 추천 계산이 가능한지 확인
- `/nearby`에서 지도 또는 샘플 리스트가 보이는지 확인
- `/nearby/[id]`에서 식당 상세와 대표 메뉴가 보이는지 확인
- `/nearby/[id]/arrived`에서 도착 확인이 가능한지 확인
- `/feed`에서 공개 베타 피드가 보이는지 확인
- `/history`에서 기록 보관함이 보이는지 확인
- `/history/[id]`에서 기록 상세가 열리는지 확인
- `/insights`에서 최소 분석 리포트가 보이는지 확인
- `/mypage`에서 온보딩 재진입과 익명 ID 초기화가 가능한지 확인

## 2. 필수 환경 변수

- `NEXT_PUBLIC_SUPABASE_URL`
- `NEXT_PUBLIC_SUPABASE_ANON_KEY`
- `SUPABASE_SERVICE_ROLE_KEY`
- `OPENAI_API_KEY`

선택:

- `NEXT_PUBLIC_KAKAO_MAPS_APP_KEY`

## 3. 베타 모드 전제

- 로그인 없이 사용 가능
- 익명 ID 기반 저장
- 클라우드 저장 우선
- 베타 피드는 공개 목록처럼 동작
- 추천 저장 결과도 같은 기록 테이블에 저장됨

## 4. 데이터 확인 포인트

- `user_meal_logs`에 스캔 저장 결과가 들어가는지 확인
- `user_meal_logs`에 추천 저장 결과가 들어가는지 확인
- `image_analysis_logs`에 분석 로그가 남는지 확인
- 썸네일이 필요한 경우 `food-images` 버킷 signed URL이 정상 생성되는지 확인

## 5. UX 확인 포인트

- 하단 FAB로 스캔 진입이 자연스러운지 확인
- 스캔 결과에서 `이대로 기록하기`, `주변 탐색`이 잘 보이는지 확인
- 추천 화면에서 `저장하기`, `주변 탐색`이 잘 보이는지 확인
- 추천 화면 안에서 탐색 미리보기(거리/영업시간/가격대)가 보이는지 확인
- 식당 상세에서 `도착 확인으로 이동` CTA가 명확한지 확인
- 피드 카드에서 기록 출처가 `사진 기록` 또는 `추천 저장`으로 보이는지 확인
- 이미지가 없는 추천 저장 기록도 깨지지 않고 보이는지 확인
- `/feed`와 `/history`의 역할이 혼동되지 않는지 확인

## 6. 테스트 시나리오

### 시나리오 A. 사진 기록

1. 홈 진입
2. 식사 선택
3. 사진 업로드
4. 분석 결과 수정
5. 저장
6. 피드 확인
7. 상세 확인

### 시나리오 B. 추천 저장

1. `/meal` 진입
2. 기준 끼니 선택
3. 메뉴 검색
4. 추천 계산
5. 추천안 선택
6. 저장
7. 피드 확인
8. 상세 확인

### 시나리오 C. 탐색 연결

1. 스캔 결과 또는 추천 결과에서 `/nearby` 이동
2. 리스트에서 식당 선택
3. 식당 상세 이동
4. 도착 확인
5. 피드 또는 분석으로 이동

### 시나리오 D. 분석 리포트

1. 기록 또는 추천 저장 몇 건 생성
2. `/insights` 진입
3. 총 기록 수, 끼니 분포, 저장 방식이 정상 노출되는지 확인
4. 최근 기록에서 상세로 이동 가능한지 확인

## 7. 알려진 운영 리스크

- 익명 ID는 브라우저 localStorage 기준이라 브라우저를 바꾸면 다른 사용자처럼 보임
- 베타 피드는 사실상 공개 목록처럼 동작하므로 민감한 데이터 업로드에 주의가 필요함
- 지도는 카카오 맵 키가 없으면 리스트 중심 fallback 흐름으로 사용됨
- 추천 저장은 이미지가 없으므로 피드에서 텍스트 카드 중심으로 보임

```bash
npm run typecheck

```

## 9. 운영 권장사항

- 베타 공유 전 피드에 테스트 데이터 몇 건을 미리 넣어두기
- `/feed`에는 공개 가능한 테스트 데이터만 노출되도록 주의하기
- 기록에 민감한 사진을 업로드하지 않도록 사전 안내하기
- 피드에 이상한 데이터가 보이면 `user_meal_logs`와 `image_analysis_logs`를 함께 확인하기
- 베타 참여자에게는 `마이 > 테스터 ID 초기화`가 있다는 점을 알려주기
